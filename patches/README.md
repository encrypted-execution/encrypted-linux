## Patch conventions

- Patches generated by `git format-patch` from a local clone of each
  upstream.
- Patch header has a `Subject:` line, a one-paragraph rationale, and a
  reference back to the relevant plan/research dossier.
- Patches are minimal: no whitespace fixups, no unrelated refactors,
  no upstream-style cleanups outside the scrambling change.
- Patches are tested against the exact upstream tag named in their
  filename (e.g. `scramble-gcc-v0.patch` applies to `releases/gcc-14`).

## How to apply

The Buildroot integration (`buildroot/external/encrypted-linux/`)
points at these patches via the standard
`<pkg>_PATCH = file://path/to/patch.patch` mechanism. Manual
application:

```
cd /path/to/upstream/checkout
git apply /path/to/encrypted-linux/patches/<name>.patch
```
